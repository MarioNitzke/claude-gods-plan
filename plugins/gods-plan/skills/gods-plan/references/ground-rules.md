# Ground Rules

These are the backbone of the process — skip them and the whole thing falls apart.

**No plan without confession.** You need at least 8 questions answered before writing anything. The plan can't exist before you've done the homework. Even if the task seems obvious, confession reveals what you don't know.

**Always get external eyes.** Send the plan to Codex and Gemini (if available), even when it looks solid. Different models catch different things — that's the whole point.

**Verify before you trust.** Don't just add what external models suggest. Check their claims against the actual code first. They don't have access to your codebase — you do.

**Let the user decide.** When something's controversial or you're unsure, ask. Don't make that call yourself.

## When to restart a phase

Sometimes a later phase shows that something earlier was off. That's fine — go back with user confirmation:
- Wrong approach → restart Phase 0 (Brainstorming)
- Missing requirements → restart Phase 2 (Confession)
- Plan structure doesn't work → restart Phase 3 (Draft)

## Watch out for these traps

| If you're thinking... | ...do this instead |
|---|---|
| "This is simple, I can skip the questions" | The user chose God's Plan for a reason. Do the full confession. |
| "I already know what to build" | Confession is about finding what you *don't* know. Ask anyway. |
| "External feedback isn't helpful here" | Show it to the user — they decide what's useful, not you. |
| "The plan is good enough, skip replan" | Replan consistently finds things. Always run it. |
| "I'll just use Glob/Grep for the analysis" | Reach for Serena first if available — deeper insights, fewer tokens. |
